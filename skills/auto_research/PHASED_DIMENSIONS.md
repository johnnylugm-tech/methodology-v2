## 核心理念

每個 Phase 的**滿分都是 100 分**，方便跨 Phase 比較。
分數 = 活躍維度的**簡單平均**（不是加權平均）。

---

## 維度分類

| 類別 | 說明 | Phase 3 | Phase 4 | Phase 5 | Phase 6+ |
|------|------|--------|---------|---------|----------|
| **MUST** | 必須納入 | D1, D5 | D1, D5 | 全部 | 全部 |
| **CAN** | 可選納入 | D6, D7 | D6, D7 | - | - |
| **PARTIAL** | 部分支援 | D8, D9 | D2, D3, D4 | D8, D9 | - |
| **INACTIVE** | 不納入 | D2, D3, D4 | D8, D9 | - | - |

### MUST/CAN 差異

| 類別 | 對分數影響 | 說明 |
|------|-----------|------|
| **MUST** | 100% 計入 | 專案必須通過的維度 |
| **CAN** | 100% 計入 | 可選，但計入時占分 |
| **PARTIAL** | 50% 計入 | 初期不完整，分數打折 |
| **INACTIVE** | 不計入 | 該 Phase 尚未支援 |

```python
score = (D1 + D2 + D3 + D4 + D5 + D6 + D7 + D8 + D9) / 9 × 100
```

## 及格/目標標準

| 標準 | 分數 | 意義 |
|------|------|------|
| **及格** | 70 分 | 最低可接受品質 |
| **目標** | 85 分 | 優秀品質 |
| **卓越** | 95 分 | 接近完美 |

### 決策規則

```
分數 ≥ 85 → ✅ 通過，進入下一 Phase
70 ≤ 分數 < 85 → ⚠️ 警告，需要改善
分數 < 70 → ❌ 不及格，Block 進入下一 Phase
```

## 實例計算：tts-kokoro-v613 (Phase 3)

### 實際分數
| 維度 | 分數 | 計入 |
|------|------|------|
| D1 Linting | 90% | ✅ |
| D5 Complexity | 40% | ✅ |
| D6 Architecture | 70% | ✅ |
| D7 Readability | 100% | ✅ |

### 計算
```
分數 = (90 + 40 + 70 + 100) / 4 × 100
     = 300 / 4 × 100
     = 75 分
```

### 結果

| 指標 | 數值 | 狀態 |
|------|------|------|
| **Phase 3 總分** | **75 分** | ⚠️ 及格但低於目標 |
| 及格線 | 70 分 | ✅ 通過 |
| 目標線 | 85 分 | ❌ 未達標 |

**瓶頸**: D5 Complexity (40%)

