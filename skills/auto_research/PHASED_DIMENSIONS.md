## 評分邏輯（v3.0）

### 原則

```
1. 分數只是測量工具，不是目標
2. 目標：所有活躍維度都「沒有已知問題」
3. 發現問題就修，不考慮「加分多少」
4. 修復後重新測量，直到維度達標
```

### 及格/目標標準

| 標準 | 分數 | 意義 |
|------|------|------|
| **及格** | 70 分 | 最低可接受，允許少數邊緣問題 |
| **目標** | 85 分 | 優秀，問題解決質量達標 |
| **卓越** | 95 分 | 接近完美 |

### 決策規則

```
對每個活躍維度：
1. 測量分數
2. 如果分數 < 85 → 找出問題
3. 修復問題
4. 回到步驟 1，直到分數 ≥ 85

所有維度都 ≥ 85 → 完成
```

```python
score = (D1 + D2 + D3 + D4 + D5 + D6 + D7 + D8 + D9) / 9 × 100
```

## AutoResearch Loop 流程（v3.0）

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 測量所有活躍維度                                          │
│    python3 dashboard.py --project /path                    │
│    ↓                                                        │
│ 2. 識別分數 < 85 的維度                                      │
│    ↓                                                        │
│ 3. 對每個低分維度：                                          │
│    ├─ 找出具體問題                                          │
│    ├─ 修復問題（不考慮加分多少）                             │
│    └─ 重新測量                                              │
│    ↓                                                        │
│ 4. 所有維度都 ≥ 85 → 完成                                   │
│    有維度仍 < 85 → 繼續修復                                  │
└─────────────────────────────────────────────────────────────┘
```

### 迭代策略（v3.0）

```
不再考慮「加分多少」

只考慮：
1. 這個維度有問題嗎？→ 有 → 修
2. 這個維度分數 < 85？→ 是 → 修
3. 修完重新測量

優先級：
- 所有有問題的維度都是最高優先
- 沒有「加分多就先修」的概念
```
