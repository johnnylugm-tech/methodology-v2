## 1. Overview

### 1.1 Purpose

Feature #11 為 methodology-v2 框架集成 Langfuse 觀測性平台，實現 OTel（OpenTelemetry）原生的分散式追蹤（distributed tracing）。所有 AI 決策節點（UAF、CLAP、風險評估、HITL）的關鍵欄位均以 OTel span attributes 形式記錄，供 Langfuse UI 和審計 API 查詢。

### 1.2 Problem Statement

methodology-v2 Framework 在 Phase 6/7 決策門涉及多個 AI 判斷節點：

- **UAF Score**：User Authorization Factor 分數
- **CLAP Flag**：Content Legitimacy Assessment Protocol 標記
- **Risk Score**：風險評估分數
- **HITL Gate**：Human-In-The-Loop 閘門決策
- **Human Decision**：人工覆核結論
- **Decided By**：最終決策者（AI Agent / Human）
- **Compliance Tags**：法規合規標籤組

這些欄位目前分散在多個模組中，無法統一查詢、趨勢分析或審計追溯。需要一個統一的 observability layer。

### 1.3 Scope

| Dimension | Description |
|-----------|-------------|
| Layer | 4 - Executive Assurance (observability) |
| Phase Integration | Phase 6 (Risk Management) / Phase 7 (Decision Gate) |
| Output | Langfuse trace spans with required attributes |
| Dashboard | Real-time view, trend charts, audit log |

## 2. Functional Requirements

### FR-11-01: Langfuse Client Initialization

- Initialize Langfuse client with OpenTelemetry (OTel) SDK
- Support configurable public/private key pair
- Support self-hosted Langfuse server mode
- Auto-flush on process exit (atexit handler)

**Inputs:**
- `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY` env vars (cloud mode)
- `LANGFUSE_HOST` env var (self-hosted URL)
- `OTEL_EXPORTER_OTLP_ENDPOINT` env var

**Outputs:**
- Initialized Langfuse client singleton
- OTel TracerProvider registered with global registry

---

### FR-11-02: OTel Span with Required Fields

Every AI decision point must emit an OTel span containing these **7 required attributes**:

| Attribute | Type | Description | Example |
|-----------|------|-------------|---------|
| `uaf_score` | float | User Authorization Factor (0.0–1.0) | `0.85` |
| `clap_flag` | bool | Content Legitimacy Assessment Protocol result | `true` |
| `risk_score` | float | Aggregated risk score (0.0–1.0) | `0.32` |
| `hitl_gate` | str | HITL gate status: `pass` / `review` / `block` | `"review"` |
| `human_decision` | str or null | Human override decision if applicable | `"approved"` |
| `decided_by` | str | Decision authority: `agent` / `human` / `system` | `"agent"` |
| `compliance_tags` | list[str] | Regulatory compliance tags | `["GDPR Art.22", "SOX"]` |

**Additional optional attributes:**
- `trace_id`, `span_id` (auto-generated by OTel)
- `model_name`, `model_version`
- `token_count`, `latency_ms`
- `session_id`, `user_id`

- Propagate W3C TraceContext across async boundaries (async/await)
- Propagate through LangChain/LangGraph node boundaries
- Propagate through HTTP headers for downstream service calls
- Support trace stitching across multi-agent sessions

### FR-11-04: Decorator-Based Auto-Instrumentation

Provide decorators for zero-boilerplate instrumentation:

```python
@observe_llm_call(model_name="claude-3-opus")
async def some_llm_invocation(prompt: str) -> str:
    ...

@observe_risk评估(point="pre_decision_gate")
async def risk_evaluation(state: dict) -> dict:
    ...
```

**Decorators:**
- `@observe_llm_call`: wraps LLM invocations, auto-captures tokens/latency
- `@observe_decision_point`: marks a decision node, emits all 7 required fields
- `@observe_tool_call`: wraps tool invocations
- `@observe_span(name)`: generic span wrapper

### FR-11-05: Langfuse Dashboard — Real-Time View

- Live view of recent traces (last 100 spans)
- Filter by `hitl_gate` status (`pass` / `review` / `block`)
- Filter by `decided_by` (`agent` / `human`)
- Filter by `compliance_tags` (multi-select)
- Show `uaf_score`, `risk_score` distribution as histogram

---

### FR-11-06: Langfuse Dashboard — Trend Charts

- Time-series of `risk_score` over rolling 7-day window
- Time-series of `hitl_gate` status distribution (pass/review/block ratio)
- HITL escalation rate: `% of spans requiring human review`
- Decision authority breakdown: `agent` vs `human` vs `system`

**Chart types:**
- Line chart: score trends
- Stacked bar: decision authority over time
- Heatmap: risk_score × compliance_tag correlation

---

### FR-11-07: Langfuse Dashboard — Audit Log

- Immutable audit log of all `human_decision` events
- Each audit entry contains: timestamp, span_id, trace_id, input_snapshot, human_decision, decided_by, compliance_tags
- Export to CSV/JSON for external compliance systems
- Retention: 90 days default (configurable)

---

### FR-11-08: Configuration Management

- All Langfuse/OTel settings via `config.py` module
- Support environment variable override
- Validate required env vars on startup; fail fast with clear error message
- Support `trace_sampling_rate` (0.0–1.0): sample only a fraction of spans in production

## 3. Non-Functional Requirements

### NFR-11-01: Performance

- Overhead per span creation: < 1ms (local in-process tracer)
- Async-safe: no blocking calls in async contexts
- Batch export: spans buffered and flushed every 5 seconds or 100 spans (whichever first)

### NFR-11-02: Security

- No PII stored in span attributes (uaf_score, clap_flag, etc. are scores/flags only)
- Langfuse API keys never logged or exposed in span attributes
- TLS required for all Langfuse cloud communication

### NFR-11-03: Compliance

- Audit log entries are append-only (no update/delete API)
- Support for compliance tags: `GDPR Art.22`, `SOX`, `HIPAA`, `CCPA`
- Data retention configurable per tag category

### NFR-11-04: Reliability

- Graceful degradation: if Langfuse is unreachable, traces are dropped (not blocking)
- OTel SDK handles retries with exponential backoff
- Local buffer overflow protection: max 10,000 pending spans

## 7. Glossary

| Term | Definition |
|------|------------|
| OTel | OpenTelemetry — vendor-neutral observability standard |
| Span | A single unit of trace; a named, timed operation |
| Trace | A collection of spans forming a directed acyclic graph |
| W3C TraceContext | Standard HTTP header format for trace propagation |
| HITL | Human-In-The-Loop; requires human approval before proceeding |
| UAF | User Authorization Factor |
| CLAP | Content Legitimacy Assessment Protocol |
