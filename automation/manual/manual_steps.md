## 目前無法補的缺口

| 缺口 | 原因 | 替代方案 |
|------|------|----------|
| 實際測試執行 | 環境限制，無法自動執行 pytest | 手動執行或 CI/CD |
| 領域知識動態載入 | 需要維護多領域模板 | 手動選擇模板 |
| 異步等待 callback | OpenClaw API 限制 | 使用 polling 模擬 |
| Quality Gate 標準化輸出 | 需要修改 doc_checker.py 源碼 | 使用簡易解析 + 人工確認 |

## 使用自動化腳本前的準備

1. **確認需求來源**：準備好使用者需求（PDF/文件）
2. **選擇專案領域**：TTS/Web/API/其他
3. **確認 Phase 數量**：只要 Phase 1-2？還是全部？

4. **手動執行的項目**：
   - Phase 1: 初始化 SPEC_TRACKING.md
   - Phase 2: 選擇架構和技術選型
