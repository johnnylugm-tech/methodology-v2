# A/B 協作觸發時機 - 強制執行表

**版本**: v1.0  
**生效日期**: 2026-03-27  
**狀態**: 強制執行（主代理無法選擇性跳過）

---

## ⚠️ 核心原則

| 原則 | 說明 |
|------|------|
| **主代理角色** | 只能「審查」，不能「執行」 |
| **Sub-agent 角色** | 執行實際工作（寫 code/測試） |
| **不能自己審自己** | Developer 不能審自己的產出 |
| **無法跳過** | 觸發條件滿足就執行，不能選擇性忽略 |

---

## 8 個 Phase 觸發規則

| Phase | 觸發時機 | 觸發條件 | 主代理動作 | Sub-agent 動作 |
|-------|----------|----------|-----------|----------------|
| **Phase 1** | 規格完成後 | 完成 SPEC.md 撰寫 | 審查規格 | 提交規格草稿 |
| **Phase 2** | 架構完成後 | 完成 ARCHITECTURE.md 撰寫 | 審查架構 | 提交架構設計 |
| **Phase 3** | 代碼完成後 | 完成 implement/ 程式碼撰寫 | 審查代碼 | 提交程式碼 |
| **Phase 4** | 測試完成後 | 完成 test/ 測試案例撰寫 | 審查測試 | 提交測試案例 |
| **Phase 5** | 驗證完成後 | 執行 pytest 結束 | 審查驗證結果 | 執行測試 |
| **Phase 6** | 報告完成後 | 完成 DELIVERY_REPORT.md | 審查報告 | 提交交付報告 |
| **Phase 7** | 風險評估完成後 | 完成 RISK_REGISTER.md | 審查風險 | 提交風險評估 |
| **Phase 8** | 配置完成後 | 完成 DEPLOYMENT.md | 審查部署 | 提交部署配置 |

## 觸發公式（主代理無法跳過）

```
IF Sub-agent 完成交付 THEN
    觸發 A/B 審查流程
    主代理角色 = Reviewer（只能審，不能做）
    IF 審查不通過 THEN
        回 Sub-agent 修正
        重新觸發審查
    END
END
```

## 主代理禁用行為

| 禁用行為 | 說明 |
|----------|------|
| ❌ 自己寫 code | 只能審查 Sub-agent 產出 |
| ❌ 自己執行測試 | 只能審查 Sub-agent 測試結果 |
| ❌ 自己寫規格 | 只能審查 Sub-agent 規格 |
| ❌ 跳過審查 | 每次 Phase 必須審查 |
| ❌ 選擇性通過 | 不能因為「趕時間」跳過 |
| ❌ 自己審自己 | Sub-agent 不能是主代理分身 |

## 驗證腳本（防止跳過）

```python
# 每個 Phase 結束後自動觸發
def verify_phase_completion(phase: int, sub_agent_output: dict):
    """驗證 Phase 完成，並觸發 A/B 審查"""
    
    # 1. 檢查 Sub-agent 確實交付
    if not sub_agent_output:
        raise Exception(f"Phase {phase}: Sub-agent 未完成交付")
    
    # 2. 檢查主代理是否執行了審查
    if not sub_agent_output.get("review_approved"):
        raise Exception(f"Phase {phase}: 缺少審查批准")
    
    # 3. 檢查審查者不是執行者
    if sub_agent_output.get("author") == sub_agent_output.get("reviewer"):
        raise Exception(f"Phase {phase}: 自己審自己，禁止")
    
    return True
```
