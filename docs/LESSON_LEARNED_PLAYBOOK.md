## 概述

---

## 6 大核心教訓

### 1. 開發流程

| 錯誤做法 | 正確做法 |
|---------|---------|
| 先開發後補文件 | **先制定 spec，再開發** |
| 同時改多個變數 | **基準要一致** |
| 一次改多個功能 | **變因要單一** |

**關鍵原則**：每次開發只改變一個變數，確保可以精確追蹤問題來源。

### 2. Constitution 使用

| 錯誤做法 | 正確做法 |
|---------|---------|
| 有 Constitution 檔案等於有使用 | **要真的執行 CLI** |
| 有文件等於有使用 | **要在開發中實際應用** |
| 使用舊版本 | **確認使用最新版本 v5.21+** |

**關鍵原則**：
```bash
# 每次開發前確認版本
python3 cli.py constitution thresholds

# 持續使用 Constitution CLI
python3 cli.py constitution check
```

### 3. 外部 Spec

| 錯誤做法 | 正確做法 |
|---------|---------|
| 不同版本用不同 spec | **基準要一致** |
| 任意分離模組 | **模組分離來自 spec** |

**關鍵原則**：外部 spec 是單一真相來源，所有版本套用相同標準。

```
└─────────────────────┬───────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 外部 spec → 制定規格                               │
│     - 收集所有需求                                      │
│     - 建立統一的 spec 文件                               │
└─────────────────────┬───────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 開發 → 遵從 spec                                 │
│     - 一次只改一個變數                                  │
│     - 保持基準一致                                      │
└─────────────────────┬───────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────────┐
│  4. Quality Gate → 把關                                │
│     - 使用 AutoQualityGate 檢查                         │
│     - 確保通過閾值                                      │
└─────────────────────┬───────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────────┐
│  5. 交付 → 測試通過                                   │
│     - 所有測試通過                                      │
│     - 文件齊全                                          │
└─────────────────────────────────────────────────────────────┘
```

## 結論

> **照著 methodology-v2 執行最佳！**

### 檢查清單

- [ ] 使用 Constitution CLI（`python3 cli.py constitution ...`）
- [ ] 套用外部 spec（統一的標準）
- [ ] Quality Gate 把關（`auto_quality_gate.py`）
- [ ] 變因單一（一次只改一個）
- [ ] 版本確認（使用 v5.21+）
