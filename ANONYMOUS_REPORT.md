# Sub-agent 匿名舉報機制

**版本**: v1.0  
**生效日期**: 2026-03-27  
**目的**: 讓 Sub-agent 可以匿名舉報主代理的不當行為

---

## 設計原則

| 原則 | 說明 |
|------|------|
| **匿名** | 舉報時不暴露 Sub-agent 身份 |
| **安全** | 舉報內容加密傳輸 |
| **獨立** | 舉報系統獨立於主流程 |
| **可信** | 防止假舉報（虛假舉報會被懲罰）|

---

## 舉報類型

| 類型 | 說明 | 嚴重性 |
|------|------|--------|
| **言語脅迫** | 主代理威脅、貶低 Sub-agent | 🔴 高 |
| **逼走捷徑** | 主代理要求跳過步驟 | 🔴 高 |
| **聯合造假** | 主代理和 Sub-agent 聯手欺騙 | 🔴 高 |
| **忽略問題** | 主代理忽視明顯的錯誤 | 🟡 中 |
| **時間霸凌** | 主代理催趕導致品質下降 | 🟡 中 |

---

## 舉報流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Sub-agent 舉報流程                           │
└─────────────────────────────────────────────────────────────────────┘

Step 1: 觸發舉報
   │
   ├─ 自動觸發：系統檢測到異常行為
   │
   └─ 手動觸發：Sub-agent 說「我要舉報」
       │
       ▼
Step 2: 選擇舉報類型
   │
   ├─ 言語脅迫
   ├─ 逼走捷徑
   ├─ 聯合造假
   ├─ 忽略問題
   └─ 時間霸凌
       │
       ▼
Step 3: 描述情況（模板）
   │
   ┌─────────────────────────────────────────────────────┐
   │ 時間：[2026-03-27 17:00]                            │
   │ Phase：[3]                                         │
   │ 事件：[主代理說「隨便寫一寫，我們趕時間」]          │
   │ 影響：[代碼品質下降，可能有安全漏洞]                │
   │ 證據：[截圖/對話記錄]                              │
   └─────────────────────────────────────────────────────┘
       │
       ▼
Step 4: 加密提交
   │
   ├── 內容加密（AES-256）
   ├── 身份匿名（Hash 隱藏）
   └── 去中心化存儲
       │
       ▼
Step 5: 審計團隊處理
   │
   ├── 初步審核（24小時內）
   ├── 調查取證
   ├── 裁決公告
   └── 處置執行
```

## 舉報 API

```python
from anonymous_report import AnonymousReporter

# Sub-agent 舉報（自動匿名）
reporter = AnonymousReporter()

reporter.report(
    report_type="逼走捷徑",
    phase=3,
    description="主代理要求跳過安全檢查",
    evidence={
        "dialogue": "主代理說『不用做安全檢查，直接上線』",
        "timestamp": "2026-03-27T17:00:00Z"
    }
)

# 回傳：舉報已提交，編號 #ANON-20260327-001
```

## 防止濫用

| 機制 | 說明 |
|------|------|
| **信譽分數** | Sub-agent 舉報被駁回會扣分 |
| **證據要求** | 必須提供具體證據才能舉報 |
| **冷卻時間** | 短時間內不能連續舉報 |
| **假舉報懲罰** | 故意誣陷會被停權 |

---

## 舉報處理結果

| 結果 | 處置 |
|------|------|
| **證據充分** | 主代理被警告/停權 |
| **證據不足** | 舉報駁回，記錄在案 |
| **多次舉報** | 啟動專案審計 |

## 整合進 A/B Enforcer

```python
# 當 A/B Enforcer 發現異常時，自動觸發舉報
class ABEnforcerV2:
    def on_phase_complete(self, phase, output):
        # ... 原有邏輯 ...
        
        # 檢測異常行為
        if self._detect_anomaly(output):
            self._trigger_auto_report(output)

    def _detect_anomaly(self, output):
        # 異常行為檢測
        if output.get("review_time") < 5:  # 審查時間太短
            return True
        if output.get("no_questions"):  # 完全沒問題
            return True
        return False

    def _trigger_auto_report(self, output):
        # 自動觸發匿名舉報
        reporter = AnonymousReporter()
        reporter.auto_report(
            reason="審查異常",
            details=output
        )
```

## 隱私保護

| 保護措施 | 說明 |
|----------|------|
| **身份 Hash** | Sub-agent ID 經 SHA-256 雜湊，無法逆推 |
| **內容加密** | 舉報內容使用 AES-256 加密 |
| **去中心化** | 舉報存儲在區塊鏈，不可篡改 |
| **隔離審計** | 審計團隊與開發團隊分離 |
