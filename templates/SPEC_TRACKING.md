## 📖 使用說明

### 1. 建立對照表

```bash
# 複製模板到專案根目錄
cp .methodology/templates/SPEC_TRACKING.md ./
```

### 2. 對照步驟

1. **閱讀規格書**：標記每個要求及其頁碼
2. **對應實作**：找到對應的程式碼位置
3. **記錄狀態**：標記完成/待處理/未實現
4. **定期更新**：每次變更後更新本表

### 3. 審查時使用

- Review 前：檢查本表確認無遺漏
- Debug 時：快速定位規格對應位置
- 交付前：確認所有 ✅ 項目已完成

## 🎯 自動化驗證

結合 `scripts/verify_spec_compliance.py` 使用：

```bash
python .methodology/scripts/verify_spec_compliance.py ./
```

會自動檢查：
- 檔案是否存在
- 關鍵邏輯是否實現
- 預設值是否優化
